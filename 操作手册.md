# 操作手册

这个手册说明以后怎么用程序排班，以及程序是按什么规则决定谁要来。

## 相关文件

- 规则总表: `ローテーション作成ルール.md`
- 排班程序: `rotation_scheduler.py`
- 示例输入: `rotation_scheduler.sample.json`
- 程序说明: `rotation_scheduler.md`
- 月表: `5月担当表.md`

## 程序会做什么

输入某一天每个人的出席情况后，程序会自动决定：

- 那天开哪些班
- 每个班由谁负责
- 哪些 `maybe / △` 的人需要补进来
- 哪些 `meeting_only` 的人只参加总会、不排课
- 王来的周是否拆成两个入门班

## 程序使用的固定规则

程序按 `わをんのローテーションについて2026.pdf` 和 `ローテーション作成ルール.md` 工作。

主要规则：

- 每个人只能负责自己固定可担当的班级
- 总会日优先远方的人：`岡本`、`岡崎`
- `今村` 如果人数够，尽量不排
- `maybe / △` 的人只有在人数不够时才加入
- `meeting_only` 只参加总会，不进班级
- 王按“这个月第几周”判断出席，不是按日期单双数判断
- 王来的周，如果人数够，就拆成 `入門(denji)` 和 `入門(王)`
- 如果人数不够，就只保留一个 `入門`

## 输入格式

程序吃一个 JSON 文件，例如：

```json
{
  "date": "2026-05-09",
  "week_index": 2,
  "meeting": true,
  "attendance": {
    "okamoto": "meeting_only",
    "Okazaki": "yes",
    "emi": "yes",
    "imamura": "maybe",
    "seityan": "meeting_only",
    "yuko": "no",
    "pei": "yes",
    "門馬": "no"
  }
}
```

### 字段说明

- `date`: 日期显示用
- `week_index`: 这个月第几周
- `meeting`: 这一天是不是总会日
- `attendance`: 每个人的出席状态

### attendance 可用值

- `yes`: 能来，也可排课
- `no`: 不来
- `maybe`: 不确定，只有人数不够时才补进
- `meeting_only`: 只参加总会，不排课

## 运行方法

在当前目录运行：

```bash
python rotation_scheduler.py rotation_scheduler.sample.json
```

如果想看原始 JSON 结果：

```bash
python rotation_scheduler.py rotation_scheduler.sample.json --json
```

## 程序输出怎么理解

程序会输出：

- 每个班最后由谁负责
- 当天实际需要来的人
- 哪些 `maybe` 的人被补进来
- 哪些人虽然回复了，但因为人数够所以不用来
- 是否拆了两个入门班

## 制作月表的流程

以后做月表时，按这个顺序：

1. 先看 `わをんのローテーションについて2026.pdf`
2. 再看 `ローテーション作成ルール.md`
3. 把每个人每一天的状态整理成 JSON 输入
4. 用 `rotation_scheduler.py` 跑每一天
5. 把结果写进月表
6. 人名要全部保留，即使当天没排上也要写出来

## 5月这次是怎么处理的

- `5/9` 是总会日
- `岡本` 和 `柴田` 是 `meeting_only`
- `岡崎`、`蔦尾`、`裴` 可以正常上课
- `今村` 是 `maybe / △`
- 因为只靠确定能来的人不够 4 个班，所以程序把 `今村` 补进来

- `5/16` 是王参加周，而且人数够
- 所以程序拆成 `入門(denji)` 和 `入門(王)` 两个班

- `5/23` 是王不参加周
- 所以只保留一个 `入門`

- `5/30` 是王参加周，但人数不够
- `岡本` 是 `△ / maybe`
- 程序把 `岡本` 补进来，但仍然只保留一个 `入門`

## 名字对应

- `okamoto` = `岡本`
- `Okazaki` = `岡崎`
- `yuko` = `相良`
- `imamura` = `今村`
- `emi` = `蔦尾`
- `seityan` = `柴田`
- `pei` = `裴`

## 后续维护

- 如果固定规则变化，先改 `ローテーション作成ルール.md`
- 如果程序逻辑变化，改 `rotation_scheduler.py`
- 如果只是某个月的出席变化，只改输入，不改程序
