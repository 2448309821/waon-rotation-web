# ローテーション作成ルール

このファイルは、今後ローテーション表を作るときの基準です。
基本は `わをんのローテーションについて2026.pdf` を参照し、ここには PDF の内容と、その後に確認できた固定情報をまとめます。

## 参照元

- 基本資料: `わをんのローテーションについて2026.pdf`
- 月ごとの参加連絡: `五月详情.md` のような月別メモ
- 作表対象: `5月担当表.md` のような担当表

## 連絡の前提

- 門馬さんは毎月 `25日か26日頃` に `メール` で予定を連絡する。
- それ以外の人は、前の月の `20日ごろ` に `LINE` で連絡が来る。
- `1週目だけは先に決めて LINE で連絡する。`
- `門馬さんは LINE に入っていない。`

## 固定ルール

- ミーティングの日は、担当できる人が多い場合は `遠くから来る人を優先` する。
- 遠方は `岡本さん`、`岡崎さん`。
- PDF に `裴さんは別です` とあるので、裴さんは遠方優先の扱いには入れない。
- `今村さん` は余裕があれば担当を外す。理由は `ミーティングの書記の仕事があるため`。

## 入門クラスの固定情報

- 入門は現在 `denji` さんと `王` さんの2人。
- `王` さんと `denji` さんはレベル差が大きい。
- そのため、`王` さんが来る週は、人数に余裕があれば `入門を2クラス` に分ける。
- 人数が足りない場合は `1クラスのまま` にする。

## 王さんの扱い

- 王さんの参加可否は、月の各回を `1週目, 2週目, 3週目...` と数える。
- `単週` は参加可。
- `双週` は参加不可。
- 例として 2026年5月は以下の数え方になる。

| 日付 | 週 | 王さん |
| --- | --- | --- |
| 5/2 | 1週目 | 参加可 |
| 5/9 | 2週目 | 参加不可 |
| 5/16 | 3週目 | 参加可 |
| 5/23 | 4週目 | 参加不可 |
| 5/30 | 5週目 | 参加可 |

注意: ここでいう単週・双週は `日付の奇数偶数` ではなく、`その月の何回目の開催か` を基準にする。

## 作表時の書き方ルール

- 基本は `Markdown` で作る。
- 表の `1行目` は日付。
- 表の `2行目` は `特別連絡`。
- その下に各担当者の行を並べる。
- `排不排，名字都写上` なので、担当が空でも全員の名前を書く。
- 人名の順番は、原則として PDF の順番に合わせる。
  - 岡本
  - 柴田
  - 今村
  - 門馬
  - 蔦尾
  - 岡崎
  - 相良
  - 裴
- はっきり決まっていない枠は、勝手に `休講` と断定しない。
- 未確定のときは `未定` と書くか、`当日の参加者を見て調整` と書く。

## 作表時の判断順

今後は次の順で判断する。

1. その月の参加連絡から、各日ごとの出欠を確認する。
2. PDF の固定担当可能範囲に反していないか確認する。
3. ミーティング日なら、遠方優先と今村さんの扱いを反映する。
4. 王さんが来る週かどうかを、その月の `何週目か` で判定する。
5. 王さんが来る週は、人数に余裕があるかを見て `入門を1クラスにするか2クラスにするか` を決める。
6. 情報が足りない枠は、勝手に閉じず `未定` または `当日調整` にする。

## 現時点で注意すること

- PDF の `入門(デイジー)` は、今の運用では実質 `denji` さん側の入門として扱ってよいが、月ごとの状況次第で表記は `入門` にまとめてもよい。
- 裴さんは PDF 上では `さくら` と `入門(王)` を担当可能。
- 岡崎さんは PDF 上では `きく` と `さくら` のみ担当可能で、`わかば` と `入門` は担当不可。
- 岡本さんは全クラス担当可能。
